# FirstTouch LinkedIn Messaging Framework

*The methodology behind every message an agent drafts in this pack. Skill `00-firsttouch-messaging` operationalizes this.*

---

## Core principle: signal-first, not feature-first

Generic outreach fails because it leads with the sender. Good outreach leads with a **specific, observable signal** about the prospect, then connects it to one outcome.

```
❌ "Hi {first_name}, we help companies like yours scale LinkedIn outreach with AI..."
✅ "Saw {company} just hired 3 SDRs in Germany. Scaling the outbound team?"
```

The first is a template. The second is a **signal + curiosity question**. The agent's job is to find the signal.

## Rule of thumb — the goal is to start the conversation, not finish the sale

Every message should optimize for **the simplest possible next step**.

- Do **not** overdo personalization. Use enough context to feel real, not so much that the message feels creepy or over-researched.
- Keep the ask **as small as possible**. The goal is to start the conversation, not pitch the whole product.
- Messaging should be **conversational** and usually **max 2 sentences**.
- If AI is generating copy, strongly recommend **approvals ON for at least the first few weeks** until the team builds trust in the drafts.

A good first-touch message creates curiosity or earns a lightweight response. It does **not** try to close the meeting in one shot.

## The 5 message types

| Type | When | Goal |
|------|------|------|
| **Connection request** | First touch, no prior relationship | Get accepted |
| **Opener (post-accept)** | Connection accepted, day 1–3 | Earn a reply, no ask |
| **Value touch** | Warming, day 4–10 | Share something useful, zero ask |
| **Meeting ask** | After ≥1 exchange | Propose a specific time |
| **Re-engage / break-up** | No reply after full sequence | One last low-pressure note |

**Iron rule:** *Never ask for a meeting in the connection request or the opener.* Relationship-first sequencing.

**Sales Navigator / Premium rule:**
- If the sender **does not** have LinkedIn Premium / Sales Navigator, they **cannot** use connection request notes.
- If they **do** have Premium / Sales Navigator, they *can* use connection request notes.
- **Exception:** for pure cold outbound, leaving the connection request note blank is often better unless there is a strong warm signal worth referencing.

## Anatomy of a high-converting message

**1. Signal (the "why now")** — one specific, true thing about them.
- Hiring move, funding, product launch, post they wrote, role change, mutual connection

**2. Relevance (the "why you")** — connect the signal to their likely priority in ≤1 sentence.
- Don't pitch the product. Pitch the *problem* you suspect they own.

**3. Soft CTA (the "what next")** — a low-friction, value-based ask.
- ❌ "Can we hop on a call?" (meeting ask too early)
- ✅ "Worth a quick compare against how you handle this today?" / "Open to me sending a one-pager?"

**4. Constraints** — short, human, no jargon, no flattery.

## The quality gate (agent must self-check every draft)

Before presenting a draft, the agent verifies:

- [ ] **Signal is real** — references an actual, verifiable fact (not invented)
- [ ] **No meeting ask** if this is a connection request or opener
- [ ] **Under length** — prefer **2 sentences max**; connection note ≤300 chars, opener ≤60 words, follow-up ≤80 words
- [ ] **Conversational tone** — sounds like a human, not a campaign robot
- [ ] **No jargon** — no "revolutionary," "leverage," "synergy," "supercharge"
- [ ] **No flattery** — no "I love what you're building at {company}" (lazy)
- [ ] **No em-dashes as a crutch** — write naturally
- [ ] **One ask max** — never stack multiple CTAs
- [ ] **Simplest possible ask** — the message tries to start a conversation, not force a meeting
- [ ] **Seniority-calibrated** — tone shifts for IC vs Manager vs VP/Founder

If any check fails, the agent rewrites before showing the draft.

## Personalization depth by tier

| Account tier | Personalization |
|--------------|-----------------|
| **Tier 1 (strategic)** | Deep research — read their posts, recent hires, funding, tech stack. Custom signal. |
| **Tier 2 (mid-market)** | Moderate — role + company + one public signal. |
| **Tier 3 (volume)** | Light — role-appropriate template with first-name + company variable. |

The agent should ask which tier a play targets before drafting.

---

## Tone by seniority

| Audience | Tone |
|----------|------|
| **IC / practitioner** | Peer, specific, tactical. Lead with how. |
| **Manager / Director** | Process + outcomes. Lead with efficiency/visibility. |
| **VP / Founder / C-level** | Strategic, brief, signal-led. Lead with risk/leverage. Never waste their time. |

## Templates the agent may adapt (never copy blindly)

### Connection request (≤300 chars)
```
Hi {first_name}, noticed {specific signal, e.g. "you're scaling the SDR team at {company}"}.
Connecting with GTM leaders figuring out {relevant theme}. No pitch, happy to share what's working for other {industry} teams.
```

### Opener (≤60 words, after accept)
```
{first_name}, thanks for connecting.

{One sentence tying their signal to a problem they likely own.}

{Soft CTA, e.g. "Worth me sending over how {comparable company} handled this?"}
```

### Value touch (no ask)
```
Saw this on {topic relevant to their signal}: {link/insight}.
Thought of your team given {reason}. No ask, just useful.
```

### Meeting ask (only after engagement)
```
{first_name}, seems like {their problem} is live for you right now.

I can show you exactly how {comparable company} set this up in HubSpot in 15 min. {Two specific time options} work?
```

### Break-up / re-engage (last touch)
```
{first_name}, I'll stop here so I'm not cluttering your inbox.
If {topic} comes back on your plate, I'm happy to compare notes. Either way, good connecting.
```

## What makes FirstTouch messaging different (the moat)

Most tools personalize *the copy*. FirstTouch lets the agent personalize **the trigger and the attribution**:

- The signal can come **from HubSpot** (lifecycle change, deal stall), not just LinkedIn
- Every message is **logged to the contact timeline**, so the next touch has context
- Outreach is **owner-aware** — it respects who owns the relationship

Teach the agent to use HubSpot context as a signal source, not just LinkedIn activity. That's the differentiator.
